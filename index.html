Abstract

Why the repository is split this way

The original upstream MINIX tree is too large to function as a readable portfolio surface. The semaphore work disappears inside that mirror unless the project is re-framed around the feature itself.

For that reason, the default branch is now only a landing document, the frozen upstream-based comparison point lives on minix-source-base, the implementation lives on minix-semaphore, and the committed rerun evidence lives on portfolio-minix-semaphore.

Contributions

Implemented scope

  1. Add Process Manager call numbers and the semaphore message ABI.
  2. Expose a small public API through minix/semaphore.h and libc wrappers.
  3. Implement counting semaphore state, delayed replies, FIFO wake-up, and interruption handling inside PM.
  4. Package the header and add test95 to exercise lifecycle, wake-up order, destroy behavior, and signal interruption.
